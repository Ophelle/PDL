Type information about C# generic types is fully preserved at runtime

The is and as operators work the same for type parameters as for any other type.

discrete classes and can be used like any other type within the generic definition.

C#
.NET generics guarantee type-safety and are verified at compile time

casts are unnecessary at runtime. Hence
generic code will run faster than non-generic (or type-erased) code that require casts when handling non-generic or type-erased objects.




Primitive and value types are allowed as type parameters in generic realizations. At runtime code will be synthesized and compiled for each unique combination of type parameters upon first use. Generics that are realized with primitive
value type do not require boxing
unboxing conversions.
